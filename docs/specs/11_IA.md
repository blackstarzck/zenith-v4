# IA (Information Architecture) — Admin UI 정보구조

## 0) 목적
- 전략 실행/모니터링/실험/리포트 업무를 빠르게 수행할 수 있도록 페이지 구조를 고정한다.
- 화면이 늘어나도 탐색 경로가 흔들리지 않도록, 내비게이션과 URL 규칙을 단일 기준으로 둔다.

참조:
- 구현 규칙: `../guides/02_DEV_GUIDE.md`
- 디자인 규칙: `../guides/04_DESIGN_GUIDE.md`
- 엔진 계약: `../architecture/06_ARCHITECTURE.md`
- 실험 규격: `10_EXPERIMENT_PROTOCOL.md`
- 화면 상세(Wireframe/UI 상호작용): `13_SCREEN_SPEC.md`
- 전체 요약 흐름: `15_ONE_PAGE_FLOW.md`

---

## 1) IA 원칙
- `strategyId`, `runId` 중심으로 정보가 이어져야 한다.
- 1차 내비게이션은 "운영(실행)"과 "분석(실험/리포트)"을 분리한다.
- 어떤 상세 화면에서도 상단에 현재 컨텍스트(`strategyId`, `runId`, `mode`, `fillModelRequested`, `fillModelApplied`)를 보여준다.
- URL은 상태를 복원 가능하게 유지한다(필터/정렬/선택된 전략 등).

중복 방지 기준:
- IA/라우트/컨텍스트 규칙의 단일 진실은 이 문서다.
- `13_SCREEN_SPEC.md`는 IA를 재정의하지 않고 화면 컴포넌트/상호작용만 다룬다.

## 2) 전역 내비게이션(Top Level)
- `운영`
- `전략`
- `실험`
- `리포트`
- `설정`

권장 라우트:
- `/runs/live`
- `/runs/history`
- `/strategies`
- `/experiments`
- `/reports`
- `/settings/exchange`
- `/settings/risk`
- `/settings/system`

---

## 3) 페이지 트리(Sitemap)
- 운영
  - 실시간 실행 대시보드: `/runs/live`
  - 실행 이력 목록: `/runs/history`
  - 실행 상세: `/runs/:runId`
- 전략
  - 전략 목록/버전: `/strategies`
  - 전략 상세: `/strategies/:strategyId`
  - 파라미터 편집: `/strategies/:strategyId/parameters`
- 실험
  - 실험 생성(조건 고정): `/experiments/new`
  - 실험 목록: `/experiments`
  - 실험 상세/비교: `/experiments/:experimentId`
- 리포트
  - run 리포트 목록: `/reports/runs`
  - run 리포트 상세: `/reports/runs/:runId`
  - 전략 비교 리포트: `/reports/compare`
- 설정
  - 거래소 연결: `/settings/exchange`
  - 공통 리스크/수수료: `/settings/risk`
  - 시스템/로그 정책: `/settings/system`

---

## 4) 핵심 사용자 플로우
1. 전략 선택/파라미터 확인
- `/strategies/:strategyId` -> `/strategies/:strategyId/parameters`

2. 실행 시작
- `/runs/live`에서 mode/fillModelRequested 확인 -> run 시작 -> `runId` 생성

3. 실행 모니터링
- `/runs/:runId`에서 이벤트/주문/체결/포지션 상태 확인

4. 실험 비교
- `/experiments/new`에서 조건 고정 -> `/experiments/:experimentId` 비교

5. 결과 검증/회귀
- `/reports/runs/:runId`에서 KPI와 `fillModelApplied`, `entryPolicy` 확인

---

## 5) 화면 컨텍스트 규칙(필수)
- 다음 값은 실행/리포트 화면 상단에 항상 노출:
  - `strategyId`
  - `runId`
  - `mode`
  - `fillModelRequested`
  - `fillModelApplied`
- 실시간 화면(`/runs/live`, `/runs/:runId`)에는 연결 컨텍스트를 추가로 항상 노출:
  - `connectionState` (`LIVE|DELAYED|RECONNECTING|ERROR|PAUSED`)
  - `lastEventAt`
- 값이 누락되면 운영 판단 오류 위험이 있으므로 릴리즈 금지.

---

## 6) URL/상태 규칙
- 목록 화면의 필터/정렬은 쿼리스트링으로 유지:
  - 예: `/reports/runs?strategyId=STRAT_A&from=2026-02-01&to=2026-03-01&sort=mdd`
- 새로고침 후에도 동일 뷰를 재현 가능해야 한다.

---

## 7) 변경 규칙
- 페이지 추가/삭제/이동 시 이 문서를 먼저 갱신한다.
- IA 변경이 UI 구조에 영향을 주면 `../guides/04_DESIGN_GUIDE.md`도 함께 갱신한다.
- IA 변경이 데이터 흐름에 영향을 주면 `../guides/02_DEV_GUIDE.md`도 함께 갱신한다.
