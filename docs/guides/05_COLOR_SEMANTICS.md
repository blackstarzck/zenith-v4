# UI 색상 시맨틱 (거래/리스크/상태)

## 목적
- 매수/매도/오류 등 주요 이벤트의 색상 규칙을 문서로 고정한다.
- 화면마다 해석이 달라지는 문제를 방지한다.

## 기본 원칙
- 색상만으로 의미를 전달하지 않는다. 텍스트, 아이콘, 툴팁을 함께 사용한다.
- 동일 의미에는 동일 색상 토큰을 사용한다.

## 거래 이벤트 색상
| 항목 | 의미 | 기본 색상 |
|---|---|---|
| BUY(매수) | 진입/매수 체결 | `#16a34a` (green) |
| SELL(매도) | 청산/매도 체결 | `#dc2626` (red) |

## 시스템/리스크 상태 색상
| 항목 | 의미 | 기본 색상 |
|---|---|---|
| SUCCESS | 정상 완료/확인 | `#16a34a` |
| WARNING | 주의/임시 경고 | `#f59e0b` |
| ERROR | 오류/실패/차단 | `#dc2626` |
| INFO | 안내/참고 | `#2563eb` |
| PAUSED | 일시정지 상태 | `#6b7280` |

## KPI 색상 규칙
| 항목 | 양수(+) | 음수(-) |
|---|---|---|
| 손익 금액/손익률 | `#16a34a` | `#dc2626` |
| MDD | 위험 지표로 고정 표시 | `#b91c1c` |

## 적용 메모
- 코인 거래의 캔들 색상(상승/하락)과 체결 이벤트 색상(BUY/SELL)은 다를 수 있다.
- 캔들 규칙이 변경되더라도 BUY/SELL 이벤트 색상은 본 문서 기준을 우선한다.
